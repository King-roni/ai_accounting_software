# Credit Note UI Spec

**Block:** out_workflow
**Layer:** 2 — Sub-Doc
**Status:** Draft

## Overview

This document specifies the UI for creating credit notes and viewing their detail. A credit note reverses all or part of a PAID invoice. The flow begins on the Invoice Detail page and produces a `credit_notes` record. Credit notes follow the status path DRAFT → ISSUED → APPLIED (optional) or VOID.

The credit note number format is `CN-YYYY-NNNN`, generated by `out_workflow.issue_credit_note` at the moment of issue (not creation). DRAFT credit notes have no public number; the UI displays a placeholder "CN-PENDING" in muted text.

## Trigger Point

The "Issue Credit Note" button appears on the Invoice Detail page (`/invoices/:invoice_id`) when `invoice_status = PAID`. It is not shown for any other status. The button label is "Issue Credit Note". Clicking it navigates to the Create Credit Note flow described below.

## Create Credit Note Flow

The flow is presented as a multi-step full-page form at `/invoices/:invoice_id/credit-note/new`. It is not a modal, because the line item selector step requires full screen width.

### Step 1 — Reason and Scope

**Original Invoice Reference** (read-only field)

Displays the source invoice number, client name, invoice total, and issue date. Non-editable. A "View Invoice" link opens the invoice in a new tab.

**Credit Reason** (required textarea)

Label: "Reason for credit note". Minimum 10 characters. Maximum 500 characters. Character counter displayed below field. No default text. Validation fires on blur and on submit.

**Credit Scope** (required toggle)

Two options presented as a radio-button toggle:

- "Full credit" — credits the entire invoice total. Selecting this disables the line item selector in Step 2 and skips to the preview.
- "Partial credit" — allows the user to specify which line items (or amounts) to credit.

Default: Full credit.

### Step 2 — Line Item Selection (Partial Credit Only)

Shown only when "Partial credit" is selected. Skipped for full credit.

A table lists all line items from the original invoice:

| Column | Notes |
|---|---|
| Description | Read-only. Truncated at 60 chars. |
| Original Qty | Read-only. |
| Original Unit Price | Read-only. |
| VAT Rate | Read-only. Inherited for credit calculations. |
| Original Line Total | Read-only. |
| Credit Qty | Number input. 0 to original qty. Defaults to 0. |
| Credit Amount | Computed: `credit_qty * unit_price`. Read-only computed field. |
| Include | Checkbox. Checking auto-sets Credit Qty to original qty; unchecking zeros Credit Qty. |

Below the table: a running total of the credit amount (excl. VAT) and credit VAT amount, and the total credit amount (incl. VAT). Updates on every input change.

Validation: at least one line item must have a Credit Qty > 0 before proceeding. Error: "Select at least one line item to credit."

### Step 3 — Preview

Displays a read-only summary of the credit note before it is created as a DRAFT:

```
Credit Note Preview
────────────────────────────────────
Original Invoice:   INV-2026-0041
Client:             ALPHA CONSULTING LTD
Credit Reason:      [reason text]
Credit Scope:       Partial / Full

Line Items Credited:
  Professional services (3 hrs)   €300.00
  VAT 19%                          €57.00
  ────────────────────────────────────────
  Total Credit Amount             €357.00

Currency: EUR
```

"Create Draft" button proceeds to DRAFT creation. "Back" returns to Step 2 (or Step 1 for full credit).

On submit: calls `out_workflow.create_credit_note`. On success: navigates to the Credit Note Detail page at `/credit-notes/:credit_note_id`.

## Credit Note Detail Page

URL: `/credit-notes/:credit_note_id`

### Page Header

| Field | Source | Notes |
|---|---|---|
| CN Number | `credit_notes.credit_note_number` | "CN-PENDING" for DRAFT in muted style |
| Status badge | `credit_notes.status` | DRAFT=neutral, ISSUED=info, APPLIED=success, VOID=muted |
| Original Invoice | `credit_notes.original_invoice_id` | Linked to invoice detail page. Format: INV-YYYY-NNNN |
| Credit Amount | `credit_notes.total_credit_amount` | With currency symbol |
| Issued Date | `credit_notes.issued_at` | Blank for DRAFT |

### Credit Note Body

**Original Invoice Reference Panel**

- Invoice number (linked)
- Client name (linked to client detail)
- Invoice total
- Invoice issue date
- Invoice payment date

**Credit Details Panel**

- Credit reason (full text)
- Credit scope: Full / Partial
- Credit amount excl. VAT
- VAT on credit (by rate, if applicable)
- Total credit amount

**Line Items Credited** (shown for partial credits only)

Same column structure as the line item selector (read-only). Shows only the credited lines.

**Issued Date** — blank for DRAFT, populated on issue.

**Applied Date** — populated when `status = APPLIED`. Shows which invoice(s) the credit was applied against.

### Action Buttons

Visibility is conditional on `credit_notes.status`.

**DRAFT**

| Button | Action | Confirmation |
|---|---|---|
| Issue | Transitions DRAFT → ISSUED. Assigns CN-YYYY-NNNN number. Calls `out_workflow.issue_credit_note`. | Yes: "Issue this credit note? The credit note number CN-YYYY-NNNN will be assigned and the client can be notified." |
| Edit | Navigate to `/credit-notes/:id/edit`. Allowed only in DRAFT. | No |
| Void | Transitions DRAFT → VOID. Requires reason. | Yes |
| Download PDF | Generates draft PDF (watermarked "DRAFT"). | No |

**ISSUED**

| Button | Action | Confirmation |
|---|---|---|
| Apply to Balance | Opens Apply Credit modal (see below). Only shown if client has outstanding unpaid invoices. | No pre-confirm |
| Void | Transitions ISSUED → VOID. Requires reason text. Not allowed if `status = APPLIED`. | Yes |
| Download PDF | Final PDF (no watermark). | No |

**APPLIED**

| Button | Action | Notes |
|---|---|---|
| Download PDF | — | No destructive actions available on APPLIED credit notes. |

**VOID**

| Button | Action | Notes |
|---|---|---|
| Download PDF | Generates void PDF (watermarked "VOID"). | No further state transitions. |

### Apply Credit Modal

Triggered by "Apply to Balance" button on ISSUED credit notes.

Lists the client's unpaid invoices (SENT, PARTIALLY_PAID, OVERDUE):

| Column | Notes |
|---|---|
| Invoice # | INV-YYYY-NNNN, linked |
| Status | Badge |
| Outstanding | Amount remaining unpaid |
| Apply Amount | Number input. Max = min(outstanding, remaining credit). |

Running total: "Credit remaining after allocation: €X.XX"

On submit: calls `out_workflow.apply_credit_note`. Creates `credit_note_allocation` record(s). Transitions credit note to APPLIED. Affected invoices re-evaluated for status (may transition SENT → PAID, PARTIALLY_PAID → PAID, etc.).

### Void Credit Note Modal

Available for DRAFT and ISSUED status only. Not available for APPLIED.

Fields:
- Void reason (required textarea, min 10 chars)
- Confirmation checkbox: "I understand this credit note will be permanently voided."

On submit: calls `out_workflow.void_credit_note`. Status transitions to VOID. The credit note detail page updates immediately with VOID badge and watermarked PDF option.

## Credit Note List

A secondary list at `/credit-notes` (linked from the sidebar under Invoices). Columns:

| Column | Notes |
|---|---|
| CN Number | Linked to detail page. "CN-PENDING" for DRAFT. |
| Original Invoice | INV-YYYY-NNNN, linked |
| Client | Client display name |
| Credit Amount | With currency |
| Status | Badge |
| Issued Date | Blank for DRAFT |

Filters: status (multi-select), date range (issued date), client (search). Default sort: created_at descending.

## Empty and Error States

- No credit notes yet: "No credit notes issued. Credit notes can be issued from a paid invoice." with a link to the invoice list.
- Original invoice not found (edge case on deep link): 404 state.
- Attempting to issue a credit note on a non-PAID invoice: API returns 422, UI shows toast "Credit notes can only be issued against paid invoices."
- Attempting to void an APPLIED credit note: button disabled; tooltip "Applied credit notes cannot be voided."

## Mobile

The Credit Note Detail page is fully readable on mobile. All tabs and sections stack vertically. CN number, status badge, original invoice link, credit amount, and issued date are visible in the header.

The Issue and Void action buttons are desktop-only. Tapping these on mobile shows a toast: "This action is only available on desktop." The Apply to Balance modal is also desktop-only for the same reason (multi-invoice allocation requires sufficient screen width for the table).

Download PDF is available on mobile and opens in the native PDF viewer.

The Create Credit Note flow (multi-step form) is desktop-only. Mobile users attempting to navigate to `/invoices/:id/credit-note/new` are shown a full-screen notice: "Creating credit notes is only available on desktop." with a button "View Invoice" to return to the invoice detail.
